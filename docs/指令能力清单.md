# Voice-Canvas 指令能力清单（需求收口）

> 本文档定义项目"计划支持哪些语音指令能力"，作为开发 backlog 与设计文档的输入。
> 每条能力标注：优先级（P0 必做 / P1 应做 / P2 选做）与实现状态（⬜ 待开发 / 🟡 进行中 / ✅ 已实现 / ❌ 未完成）。
> 分级逻辑：按语义复杂度递增（L1→L6），对应项目的深度梯度。

---

## 优先级说明

| 优先级 | 含义 | 目标 |
|---|---|---|
| **P0** | 必做，决定项目能否成立 | 两天内必须完成，演示核心闭环 |
| **P0-lite** | 必做但做最小可用版 | 实现成本低、叙事价值高，做轻量版即可（如危险指令仅需 `pendingAction`+"确定"放行，不做复杂确认系统） |
| **P1** | 应做，体现项目深度 | 时间允许必做，是评分亮点 |
| **P2** | 选做，锦上添花 | 有余力再做，未完成需写明原因 |

## L3 画布与会话控制指令（P0）

meta 操作。**正式演示路径下这些全部走语音**；侧边栏对应的鼠标按钮仅作开发期/故障恢复入口，不出现在演示中（见 §纯语音边界说明）。

**关于"暂停监听"的纯语音闭环（重要）**：真正停掉 ASR 就听不到"继续"，自相矛盾。因此采用**三档监听电源态**：

- `active`（全量解析）：正常绘图。
- `standby`（仅唤醒词）：**ASR 仍在跑、麦克风仍开**，但只对"继续 / 开始绘图"等唤醒词响应，其余转写一律忽略、不调 LLM、不画图。唤醒词在本地对 final transcript 做字符串匹配，零成本零延迟。
- `off`（硬关闭）：真正关麦克风，仅隐私场景用，需鼠标重启；**演示不用**。

"暂停"→ 进入 `standby`；"继续 / 开始绘图"→ 回到 `active`。这样纯语音闭环不破。

| 能力 | 示例语音 | 预期行为 | 优先级 | 状态 |
|---|---|---|---|---|
| 撤销 | "撤销" / "退回上一步" | 回退最近一次操作 | P0 | ✅ |
| 重做 | "重做" | 恢复被撤销的操作 | P1 | ✅ |
| 清空画布 | "清空画布" / "全部删掉" | 清空所有对象（**P0-lite 二次确认**，见 L6） | P0 | ⬜ |
| 暂停监听（→standby） | "暂停监听" / "停一下" | 进入 standby，仅听唤醒词，不画图 | P0 | ✅ |
| 继续监听（→active） | "继续" / "开始绘图" | 从 standby 恢复全量解析 | P0 | ✅ |
| 导出图片 | "保存成图片" | 导出画布为 PNG | P1 | ⬜ |
| 切换模型 | "切换到 DeepSeek" | 切换解析所用模型 | P2 | ⬜ |

## L5 复杂组合指令（P0/P1 — 最大亮点）

单条指令拆解为多个图元 / 含循环、布局、空间关系。体现"复杂指令拆解能力"。
**布局由确定性布局器落地**（LLM 输出空间意图，代码算像素）：P0 布局器覆盖 `anchor 九宫格 + row + grid + relative 四方向`；`around/circle / 复杂 align` 属 P1。

| 能力 | 示例语音 | 预期行为 | 优先级 | 状态 |
|---|---|---|---|---|
| 语义组合体 | "画一个房子" | 拆成 墙+屋顶+窗+门 一组图元（建 group），算好相对坐标 | P0 | ⬜ |
| 数量+循环 | "画三个圆" | 生成 3 个圆，row 布局 | P0 | ⬜ |
| 渐变+排列 | "画五个由小到大的圆排成一排" | 循环+尺寸递增+row 布局 | P0 | ⬜ |
| 网格排列 | "画一个 3x3 的网格圆" | grid 布局 | P0 | ⬜ |
| 相对定位 | "在房子左边画一个圆" | relative 四方向定位（参照 group/对象） | P0 | ⬜ |
| 颜色序列 | "画三个圆，红黄蓝" | 批量创建+逐个上色 | P2 | ⬜ |
| 环形布局 | "把六个圆排成一圈" | around/circle 布局 | P1 | ⬜ |
| 相对叠放 | "把圆放在方块上面" | above 相对定位 | P2 | ⬜ |
| 其他组合体 | "画一个太阳" / "画一辆车" | 拆解为图元组合（建 group） | P2 | ⬜ |

## L6 容错与澄清指令（P1 — 评分硬指标）

题目明确要求的"容错性"。核心是一套**置信度判断 + 主动澄清**协议。

| 能力 | 示例场景 | 预期行为 | 优先级 | 状态 |
|---|---|---|---|---|
| 危险指令二次确认 | "清空画布" / "全部删掉" | 进入 `pendingAction`，反问"确定清空？说『确定』继续"，说别的则取消 | P0-lite | ⬜ |
| 模糊指代澄清 | "那个东西弄红"（多个对象时） | AI 反问"你是指哪一个？" | P1 | ⬜ |
| 缺参数推断 | "画个圆"（无颜色/位置） | 用合理默认值，不报错 | P0 | ⬜ |
| 口误容忍 | "画一个元"（识别错字） | 模糊匹配到"圆" | P1 | ⬜ |
| 无法理解 | 完全无关的话 | 礼貌提示"没听懂，请再说一次" | P1 | ⬜ |
| 越界保护 | "移到屏幕外面" | 限制在画布范围内（clamp 兜底） | P2 | ⬜ |
| 非法操作 | "删掉第十个"（只有3个） | 友好提示对象不存在 | P1 | ⬜ |

## 演示脚本预设（视频用）

为保证视频有冲击力，演示按"由浅入深"展开，覆盖每个层级的亮点：

1. **基础**（L1+L2）："画一个红色的圆" → 即时出现
2. **复杂拆解**（L5）："在右边画一个房子" → 侧边栏展示拆解过程，一次生成图元组
3. **循环布局**（L5）："画五个由小到大的彩虹圆排成一排"
4. **上下文指代**（L4）："把它变大" → "把最大的那个删掉" → "刚才那个房子往左移"
5. **容错澄清**（L6）：说句模糊指令 → AI 反问 → 确认后执行
6. **纯语音证明**（L3）：全程用"暂停监听""继续"等语音控制，证明未碰鼠标键盘

## 范围边界（明确不做）

为保证两天聚焦核心，以下**明确不做**，未完成项在设计文档中如实说明：

- 多人协作 / 实时同步
- 云端存储 / 账号系统 / 作品持久化
- 自由曲线手绘（与"纯语音"理念冲突，且非重点）
- 任意自然语言的开放式理解（聚焦绘图领域 DSL）
- 多模型自由选择（仅 Claude / DeepSeek 二选一切换）
- 在线部署（本地演示 + 视频）
